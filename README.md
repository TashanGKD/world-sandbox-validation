## 项目简介

### 在大框架中的位置

「人—智能体混合数字世界」世界底座的第三层：**验证方法**。

公理体系（[world-axiom-framework](https://github.com/TashanGKD/world-axiom-framework)）定义了"世界应该是什么"，体系结构（[world-three-particle-impl](https://github.com/TashanGKD/world-three-particle-impl)）定义了"世界实际上是怎样设计和运行的"，本项目回答第三个问题：**两者在真实运行场景中是否一致、差在哪里、根因在哪层。**

**为什么不能只靠文字审查**：即使文档写得再细，不经过在具体状态场景内的显式因果推演，就无法确认：
- 在所有合法初始状态 + 事件序列下，系统是否总能到达合法终态
- G 层每条要求是否真正在 v 文档的 L2/L3/EP 层中落地
- 发现的问题根因在哪里——v 文档设计缺陷、G 层落地不完整，还是 T-PA-G 体系本身有遗漏

### 三文档逻辑关系

```
T-PA-G 规范体系（应然）  ←── 验证基准
        ↓ 验证什么
v 文档实现（实然）        ←── 被验证对象
        ↓ 按规则推演
沙盘推演（验证一致性）    ←── 本项目
        ↓ 输出结论
根因链：G层/PA层/T层      ←── 三层溯源
```

### 双重功能

1. **系统验证**：检验 v 文档实现是否满足 T-PA-G 规范的每条要求
2. **理论元验证（M4）**：当失败无法归因到任何 T-PA-G 条目时，说明理论体系本身有遗漏，需触发公理修订

## 核心方法论

### 判定分级（P1–P4）

| 判定 | 含义 | 处理 |
|------|------|------|
| **P1** 通过 | 推演路径唯一合法，终态合法 | 记录结论 |
| **P2** 暂缓 | 文档有歧义/空白，无法完成推演 | 发起文档澄清 |
| **P3** 失败 | v 文档存在明确设计缺陷，根因在 G 层 | 提 ISSUE，触发联动规则场景D |
| **P4** 升级 | 失败无法归因到 T-PA-G，理论体系有遗漏 | 启动 M4 元验证，触发联动规则场景E |

### 标准记录格式（M2 五节）

每条推演记录包含：
- **A. 元信息**：记录编号、任务类型、使用文档版本
- **B. 验证对象**：直接验证的 G/PA/T 条目
- **C. 文本依据**：引用 T-PA-G 和 v 文档的具体条款
- **D. 初始状态**：完整可复现的系统状态描述
- **E. 推演过程 + 判定**：逐步推演 + P1/P2/P3/P4 结论 + 根因链

### 六问判准（系统是否真正闭环）

```
1. 事件是否到达主体？（路由完备性）
2. 是否被真正摄取？（感知-接收链路）
3. 是否允许打断当前活动？（中断优先级）
4. 若响应，如何输出？（输出生成路径）
5. 如何写回主体状态？（写隔离与状态更新）
6. 写回后如何继续调度？（持续再进入）
```
