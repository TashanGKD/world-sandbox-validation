```

├── M2  任务与记录规范   ← 执行阶段：如何写标准记录（含 A-E 五节格式）
├── M3  总结汇总与报告   ← 收尾阶段：如何写 G→PA→T 三层结论报告
├── M4  体系缺口元验证   ← 进阶：发现 T-PA-G 体系本身遗漏时使用
├── M5  速查参考         ← 随时查阅：判定标准、状态机、术语表
└── M6  任务库起始清单   ← 480+ 个标准任务的完整清单
```

## 核心概念

**判定分级（P1–P4）**：
- **P1 通过**：场景在 v 文档规则下有唯一合法推演路径，且终态合法
- **P2 暂缓**：文档有歧义/空白，需澄清，不算失败
- **P3 失败（实现缺口）**：v 文档规则存在明确缺陷，根因可归到 G 层
- **P4 升级（理论缺口）**：失败无法归因到 G/PA/T 任何条目，需启动 M4 元验证

**三文档关系**：
- T-PA-G 规范（应然）→ v 文档实现（实然）→ 沙盘推演验证（一致性）
