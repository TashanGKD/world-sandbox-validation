## §0 关键区分：EXEC 批次 vs 判定等级（P）

> **必读！** 本文档使用两套独立的"P 编号"和"EXEC 编号"，含义完全不同，绝对不能混用。

| 编号 | 含义 | 使用场景 |
|------|------|---------|
| **EXEC-0 / EXEC-1 / EXEC-2** | **执行批次**（先跑哪批）| 任务设计时标注，指导执行顺序 |
| **P1 / P2 / P3 / P4 / P5** | **判定等级**（满足性结论）| 场景记录时填写，表达该要求是否成立 |

## §1 判定等级速查（P1–P5）

| 等级 | 含义 | 处理规则 | 能否作为最终结论 |
|------|------|---------|---------------|
| **P1** | 完全满足 | 归档 | ✅ 需 ≥E1 证据 |
| **P2** | 部分满足 | 标注薄弱点后归档 | ⚠️ 需标注"待完善" |
| **P3** | 概率性不满足 | **必须继续下钻到 P4** | ❌ 不是终点 |
| **P4** | 完全不满足 | 立即生成 RC- 根因链 | ✅ 需 ≥E1 证据 |
| **P5** | 暂不可判定 | 补充场景后重判 | ❌ P5 ≠ 没问题 |

**核心禁令**：P3 不被 P1 稀释。P5 不是中性票。关键任务未覆盖时不得给 P1。

## §2b 失败程度速查（D1–D4）

| 等级 | 含义 | 典型特征 |
|------|------|---------|
| **D1** | 局部失败 | 仅影响单个场景或单类边界条件，低传播性 |
| **D2** | 条件性失败 | 在特定条件分布下可重复出现（常为 P3）|
| **D3** | 结构性失败 | 对一类任务域持续造成影响（多场景复现）|
| **D4** | 根本性失败 | 直接破坏系统基础成立条件（指向 T1/T2/T3 硬失败）|

**D 级与修复优先级的推荐对应**：D4 → FP0/FP1；D3 → FP1/FP2；D2 → FP1（若影响高层公理）；D1 → FP2/FP3

## §2c 修复优先级速查（FP0–FP3）

| 等级 | 含义 | 典型情形 |
|------|------|---------|
| **FP0** | 不修则无法继续验证或系统基础不成立 | 双真源、非法写、无再进入链 |
| **FP1** | 高优先，显著影响系统可信性 | effect 无终局、bounded 失效、耦合失效 |
| **FP2** | 中优先，影响特定功能域或边界条件 | 某类 hazard 下失效、局部活锁 |
| **FP3** | 低优先，可先保留观察 | 文档表述模糊、统计覆盖不足 |

## §3 任务类型速查（5 类）

| 类型 | 含义 | 验证层级 | 版本升级时 | EXEC 批次 |
|------|------|---------|-----------|---------|
| **GEN** | 通用：去掉 v6 字段名仍成立 | T、PA | ✅ 无需重写 | EXEC-0 |
| **MAP** | 桥接：验证 GEN→G 映射完备性 | T/PA → G | ✅ 大部分可复用 | EXEC-0/1 |
| **SYS** | 体系绑定：依赖 v6 具体 schema | G | ❌ 需更新 | EXEC-1/2 |

**MAP 任务的独特作用**：MAP 失败 ≠ GEN 失败 ≠ SYS 失败。MAP 失败 = 抽象判据存在但体系落地映射不完整。

  G2-R007 → PA1+PA4+PA5+PA7+PA9 → T1+T3+T5

## §17 中英双语术语字典（Glossary of Terms）

### 17.1 核心目标与公理层（T & PA Layers）

| 中文原词 | 推荐英文翻译 (Term) | 英文精准定义与解释 (Definition & Explanation) |
|---------|-------------------|------------------------------------------|
| **写权唯一性** (PA5) | **Unique Write Authority**<br>*(or Single Writer Principle)* | **Definition**: Every durable state component must have exactly one authoritative writer.<br>**Explanation**: No component is allowed to bypass the owner's interface to directly mutate its state. This prevents race conditions and ensures a single source of truth for any state change. |
| **效应有终局** (PA6) | **Guaranteed Effect Resolution**<br>*(or Effect Terminality)* | **Definition**: Any pending effect (e.g., notification, obligation, lease) must transition into a terminal state within a finite time.<br>**Explanation**: Transient states cannot exist indefinitely (no "infinite pending"). They must eventually be resolved as consumed, expired, cancelled, or archived to prevent unbounded accumulation. |
| **持续再进入** (PA4) | **Guaranteed Rescheduling**<br>*(or Continuous Re-entry)* | **Definition**: The system must guarantee that an agent in any reachable state will be granted a subsequent opportunity to evolve (evaluate its rules) within a finite time.<br>**Explanation**: An agent cannot permanently "sleep" or drop out of the scheduler loop. There must always be a valid `next_wakeup` trigger. |
| **语义进展** (PA8) | **Semantic Progression** | **Definition**: The system must yield verifiable, goal-oriented advancements over time, rather than merely churning through superficial state changes.<br>**Explanation**: Differentiates true productivity (e.g., closing a task) from spurious activity (e.g., endless chatting without conclusions). |
| **存在性与可区分** (PA1) | **Agent-Environment Distinguishability** | **Definition**: The state space must formally separate the subjective state of agents from the objective state of the environment.<br>**Explanation**: Clear boundaries prevent the system from collapsing into an amorphous block of data. |
| **有界性** (PA2) | **State Boundedness** | **Definition**: Any state variable capable of unbounded growth must be constrained by a hard upper limit or a truncation mechanism.<br>**Explanation**: Prevents out-of-memory errors and history bloat (e.g., capping message histories or sleep debt). |
| **可恢复性** (PA3) | **Resource Recoverability** | **Definition**: Any consumable critical resource (like energy) must have an accessible path from depletion back to a functional level.<br>**Explanation**: Prevents permanent agent death due to resource exhaustion. |
| **信息可达性** (PA7) | **Information Reachability** | **Definition**: Events intended for a specific agent must be successfully routed to that agent's context within their validity period.<br>**Explanation**: Guarantees the input side of coupling—agents aren't blind to the world. |
| **观察—行动耦合** (PA9) | **Observation-Action Coupling** | **Definition**: An agent must be able to both perceive the environment and exert causal effects upon it, forming a closed feedback loop.<br>**Explanation**: Agents cannot be purely passive listeners or blind actors. |

### 17.2 架构与状态管理（Architecture & State Management）

| 中文原词 | 推荐英文翻译 (Term) | 英文精准定义与解释 (Definition & Explanation) |
|---------|-------------------|------------------------------------------|
| **持久化真状态** | **Durable State** | **Definition**: The definitive, persistent state of the system that must survive crash-restarts.<br>**Explanation**: Any change here must be recorded in the Global Event Log first (WriteEvent-first). |
| **运行时影子状态** | **Ephemeral State** | **Definition**: Temporary, derived, or cached state used during runtime that does not require persistent logging.<br>**Explanation**: Can be safely recalculated from Durable State upon system restart. |
| **进展投影 P(s)** | **Progress Projection** | **Definition**: A specific metric or subset of variables used to quantify true semantic advancement in a given context.<br>**Explanation**: In a task context, progress projection might equal the number of completed deliverables. |
| **迟滞锁** | **Hysteresis Lock** | **Definition**: A safety mechanism that forces an agent to remain in a recovering state until a safe resource threshold is restored, preventing rapid state toggling.<br>**Explanation**: If energy hits 0, the agent cannot take high-consuming tasks until energy reaches exactly 30, stopping micro-oscillations between 0 and 1. |
| **认知连续性** (G7) | **Cognitive Continuity** | **Definition**: The capability of an LLM-based agent to resume an interrupted context seamlessly without losing focus or working memory.<br>**Explanation**: Achieved through mechanisms like `resume_token` and `ForegroundStack` popping. |
| **动作端到端闭环** | **Action-to-Receipt Loop** | **Definition**: The complete lifecycle of an action: Intent → Emit → Route → World Apply → Action Receipt.<br>**Explanation**: An agent shouldn't assume an action succeeded until it receives cryptographic or systemic confirmation (receipt). |

### 17.3 异常、失效模式与测试（Anomalies & Failure Modes）

| 中文原词 | 推荐英文翻译 (Term) | 英文精准定义与解释 (Definition & Explanation) |
|---------|-------------------|------------------------------------------|
| **双真源 / 脑裂** | **Split-Brain / Dual Sources of Truth** | **Definition**: A critical failure where two different components claim authority over the same state, leading to irreconcilable data divergence.<br>**Explanation**: Example: Agent updates its local memory claiming a task is done, but the World state rejects it, yet both persist. |
| **记忆域污染 / 记忆串台** | **Memory Bleed** | **Definition**: The erroneous injection of episodic memory from one semantic context into a completely unrelated context.<br>**Explanation**: E.g., an agent hallucinates casual chat details into a formal code review because the `source_field_id` isolation failed. |
| **语义空转 / 假活性** | **Spurious Liveness / Semantic Idling** | **Definition**: The system exhibits continuous computational activity (e.g., sending messages) but yields zero net change in the Progress Projection.<br>**Explanation**: Looks alive in the logs, but structurally stuck. |
| **吸收态 / 僵死态** | **Absorbing State / Deadlock** | **Definition**: A state from which the system has no valid exit path or transition rule.<br>**Explanation**: E.g., two agents indefinitely waiting for each other to fulfill a mutually blocking obligation. |
| **低头族悖论** | **Phubbing Paradox** | **Definition**: A design flaw where an agent's digital focus entirely eclipses critical physical/local events.<br>**Explanation**: An agent focused on a digital group chat fails to respond when physically tapped on the shoulder in a co-located space. |
| **体系缺口** | **Framework Gap** | **Definition**: A systemic failure that cannot be attributed to a bug in implementation, but rather to a missing constraint in the underlying theoretical framework (T-PA-G).<br>**Explanation**: The rules were implemented perfectly, but the rules themselves were insufficient. |
| **强制逃离机制** | **Timeout Breaker / Escape Hatch** | **Definition**: A system-level override that forcibly terminates a livelocked or stalled interaction after N-ticks of no progress.<br>**Explanation**: Ensures that the axiom of Semantic Progression (PA8) is strictly enforced without requiring human intervention. |
