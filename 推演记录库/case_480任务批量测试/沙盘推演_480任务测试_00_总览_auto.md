### 5.1 写隔离 / G1-R001 文档歧义（唯一 P2，已记录）

| 项目 | 内容 |
|------|------|
| **现象** | v6 §11.4 规定 Agent B 在 `R_B^apply` 中更新 `/coupling/notifications/agent_i.queue.json`（surfaced / acknowledged）；v6 变量分类表（附录 A）规定该路径属于 **V_C**、由 **C 自身** 写入。 |
| **冲突** | 若实现为 B 直接写该文件，则违反 **G1-R001（写隔离）** 与 **I1（V_C 只能由 C 写入）**。 |
| **影响** | GEN-LEGAL-011（跨粒子无旁路写）、SYS-LEDGER-019（WriteEvent-first 写回）判定为 **P2（部分满足）**。 |
| **结论** | 属 **v6 文档内歧义**，非 T-PA-G 体系缺口；当前推演采纳「B 仅写自身 V_B」的最小写集，主路径满足。 |
| **建议** | v6 文档修订：明确 notification 的 surfaced/acknowledged 更新由 **C 的 R_C^apply** 根据 B 发出的事件执行；或显式将 `agent_i.queue` 划为 B 的授权写区并说明与 I1 的兼容性。 |
| **出处** | 《沙盘推演记录_v6_sonnet3_auto.md》场景二、步 8、检查点 CP3、J 节。 |
